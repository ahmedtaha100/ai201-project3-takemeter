## Exact run order to finish

```bash
python3 -m venv .venv && source .venv/bin/activate && pip install -r requirements.txt
cp .env.example .env          # then put your real GROQ_API_KEY in .env
export PYTHONPATH=$PWD/src

python src/collect.py --target 400            # 1) residential IP
python src/label.py                           # 2) needs GROQ_API_KEY -> labeled CSV + needs_review
#   --> human-review data/needs_review.csv, fix labels, re-check distribution
python src/prepare_splits.py                  # 3) one 70/15/15 split, same test set for both
python src/baseline.py                        # 4) zero-shot baseline FIRST (don't peek at fine-tuned)
python src/finetune.py                        # 5) fine-tune  (or run notebooks/takemeter.ipynb on Colab T4)
python src/evaluate_models.py                 # 6) evaluation_results.json + confusion_matrix.png + report_blocks.md
python src/error_analysis.py                  # 7) stretch: wrong_predictions.csv (+ AI failure-analysis pass)
python src/calibration.py                     # 7) stretch: calibration.png + ECE
python src/app.py                             # 7) stretch: deployed interface for the demo
```

Then paste `artifacts/report_blocks.md` into README §6, write the 3 error analyses from
`wrong_predictions.csv`, fill the sample-classifications table, and record the demo.
